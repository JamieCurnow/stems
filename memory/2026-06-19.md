## Flower availability status feature

Added a categorical `availabilityStatus` to flowers, alongside the existing
`stemsAvailable` count. Both optional/independent — grower can use either, both
or neither. Statuses: Good / Limited / Very limited / Sold out / Available
midweek / next week / soon (slugs: good, limited, very_limited, sold_out,
midweek, next_week, soon) — defined in `shared/utils/flowers.ts` with semantic
colours for badges.

Key decisions:
- Sold-out is now derived from EITHER signal (count `0` OR `sold_out` status) —
  `isSoldOut`/`isInStock` were changed to take a flower-like object. No external
  callers existed, so the signature change was safe.
- Public profile list: status / stems / price now **stack vertically** (didn't
  fit horizontally); status shows as a coloured `UBadge`. Stems line drops the
  word "available" → "120 stems". Falls back to "Available" only when there's no
  status and no count.
- Migration `0011_flower_availability_status.sql` (plain ADD COLUMN, nullable).
